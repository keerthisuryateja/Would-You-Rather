# Would You Rather: What You Need To Do

Use this checklist to make the app run with real data and deploy safely.

## 1. Local Environment

1. Copy `.env.example` to `.env`.
2. Add your Supabase project values:

```bash
VITE_SUPABASE_URL=https://YOUR_PROJECT.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR_SUPABASE_ANON_KEY
```

3. Run:

```bash
npm install
npm run dev
```

## 2. Supabase Database Setup (Required)

Run this SQL in Supabase SQL editor.

```sql
create table if not exists public.questions (
  id bigint generated by default as identity primary key,
  option_one text not null,
  option_two text not null,
  votes_one integer not null default 0,
  votes_two integer not null default 0
);

insert into public.questions (option_one, option_two)
values
  ('Travel to the past', 'Travel to the future'),
  ('Fly forever', 'Breathe underwater'),
  ('Read minds', 'Become invisible')
on conflict do nothing;

alter table public.questions enable row level security;

create policy if not exists "Allow read questions"
on public.questions
for select
using (true);

create policy if not exists "Allow update votes"
on public.questions
for update
using (true)
with check (true);

create or replace function public.increment_vote(row_id bigint, col_name text)
returns void
language plpgsql
security definer
as $$
begin
  if col_name = 'votes_one' then
    update public.questions
    set votes_one = votes_one + 1
    where id = row_id;
  elsif col_name = 'votes_two' then
    update public.questions
    set votes_two = votes_two + 1
    where id = row_id;
  else
    raise exception 'Invalid vote column';
  end if;
end;
$$;

grant execute on function public.increment_vote(bigint, text) to anon;
```

## 3. GitHub Secrets

Set these in GitHub repository settings:

- Settings -> Secrets and variables -> Actions -> Secrets
- Add repository secrets:
  - `VITE_SUPABASE_URL`
  - `VITE_SUPABASE_ANON_KEY`

What is secret vs public:

- `VITE_SUPABASE_ANON_KEY`: public in frontend apps (safe to expose in browser), but keep it out of source files for cleaner ops.
- `VITE_SUPABASE_URL`: public.
- `SUPABASE_SERVICE_ROLE_KEY`: secret, never put in frontend env (`VITE_*`), never commit it.

## 4. Verify It Is Working

1. Open app and vote on one question.
2. Refresh browser.
3. If the count stays increased, live DB writes are working.
4. Open Supabase table `questions` and verify the same row changed.

## 5. If Results Still Look Fake

Check these quickly:

1. `.env` exists and has correct values.
2. `questions` table has rows.
3. RLS policies allow `select` and `update`.
4. `increment_vote` function exists and anon can execute it.
5. Browser console has no Supabase 401/403/CORS error.

## 6. Optional Hardening

1. Add CAPTCHA or rate limiting to prevent vote spam.
2. Store one-vote-per-user rules (IP hash/session/user auth).
3. Add database constraints to avoid negative votes.
